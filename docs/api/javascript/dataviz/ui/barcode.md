# kendo.dataviz.ui.Barcode

## Configuration

### renderAs `String` *(default: "svg")*

Sets the preferred rendering engine.
If it is not supported by the browser, the Barcode will switch to the first available mode.

The supported values are:

* "canvas" - renders the widget as a Canvas element, if available.
* "svg" - renders the widget as inline SVG document, if available


<div class="meta-api-description">
Configure how the barcode is visually generated by selecting the rendering method to either draw using Canvas or as inline SVG graphics, enabling control over output format and compatibility with different browsers or environments. Choose between rendering strategies such as Canvas pixels or scalable vector graphics to optimize appearance, performance, or integration requirements, with automatic fallback to supported renderers if the preferred mode is unavailable. This setting controls whether barcodes appear as bitmap-based Canvas drawings or resolution-independent SVG shapes, useful for adapting to display preferences, export formats, or customization needs in barcode generation workflows.
</div>

#### Example - Render as SVG, if supported

    <div id="barcode"></div>
    <script>
    $("#barcode").kendoBarcode({
      width: 300,
      value:"123456",
      renderAs: "svg"
    });
    </script>

## Methods

### exportImage
Exports the barcode as an image.
The result can be saved using [kendo.saveAs](/api/javascript/kendo/methods/saveas).

The export operation is asynchronous and returns a [promise](https://api.jquery.com/Types/#Promise).
The promise will be resolved with a PNG image encoded as a [Data URI](https://developer.mozilla.org/en-US/docs/data_URIs).


<div class="meta-api-description">
Export barcode rendering as a high-quality PNG image for saving, downloading, or further manipulation by invoking a method that asynchronously produces a promise resolved with a base64-encoded Data URI of the barcode image, enabling easy integration with file saving utilities or custom image processing workflows, supporting use cases like exporting generated barcodes, converting barcode visuals to image files, or automating image capture from barcode components for storage and sharing.
</div>

#### Parameters

##### options `Object` *(optional)*
Parameters for the exported image.

##### options.width `String`
The width of the exported image. Defaults to the barcode width.

##### options.height `String`
The height of the exported image. Defaults to the barcode height.

#### Returns
`Promise` A promise that will be resolved with a PNG image encoded as a Data URI.

#### Example - Exporting a barcode to an image
    <div id="barcode"></div>
    <script>
        $("#barcode").kendoBarcode({
          value: "BAR",
          width: 300
        });

        var barcode = $("#barcode").getKendoBarcode();
        barcode.exportImage().done(function(data) {
            kendo.saveAs({
                dataURI: data,
                fileName: "barcode.png"
            });
        });
    </script>

### imageDataURL

Returns a PNG image of the barcode encoded as a [Data URL](https://developer.mozilla.org/en-US/docs/data_URIs).

> This method is deprecated and replaced by [exportImage](/api/javascript/dataviz/ui/barcode/methods/exportimage).


<div class="meta-api-description">
Generate or get a base64-encoded PNG image string of a barcode for embedding in HTML img tags, inline embedding, downloading, or uploading to servers, convert barcode visuals to data URLs for easy image handling, extract barcode graphics as encoded strings for front-end or back-end processing, enable exporting barcode images as inline data URIs usable in web apps or APIs, obtain barcode visuals in a portable, text-encoded image format suitable for embedding or transfer, encode barcode renderings into PNG data URLs for dynamic image source assignment or server-side workflows.
</div>

#### Returns

`String` A data URL with `image/png` MIME type. Will be `null` if the browser does not support the `canvas` element.

#### Example - show a snapshot of the Barcode

    <div id="barcode"></div>
    <a download="export.png" id="export" class="k-button">Export PNG</a>
    <script>
    $("#barcode").kendoBarcode({
      value: "FOO",
      width: 300
    });

    $("#export").on("click", function() {
      var barcode = $("#barcode").data("kendoBarcode");
      var imageDataURL = barcode.imageDataURL();

      if (navigator.msSaveBlob) {
        var blob = toBlob(imageDataURL, "image/png");
        navigator.msSaveBlob(blob, this.getAttribute("download"));
      } else {
        this.href = imageDataURL;
      }
    });

    // See: https://goo.gl/qlg5dd
    function toBlob(base64, type) {
      var rawData = base64.substring(base64.indexOf("base64,") + 7);
      var data = atob(rawData);
      var arr = new Uint8Array(data.length);

      for (var i = 0; i < data.length; ++i) {
        arr[i] = data.charCodeAt(i);
      }

      return new Blob([ arr.buffer ], { type: type });
    }
    </script>

Trigger or force the barcode display to update and refresh its visual rendering whenever data, configuration settings, style options, or container dimensions change; manually invoke a method to re-render the barcode dynamically after modifying values, layout, formatting, or appearance to ensure the barcode output accurately reflects the latest state, supporting use cases where automatic updates might not occur or when immediate visual refresh is needed following property adjustments or interactive changes.
</div>

#### Example

    <div id="barcode"></div>
    <script>
    $("#barcode").kendoBarcode({
      value: "FOO",
      width: 300
    });

    $("#barcode")
       .data("kendoBarcode").redraw();
    </script>

### resize

Adjusts the widget layout to match the size of the container.


<div class="meta-api-description">
Adjust or update the barcode display to fit new container dimensions by recalculating size, layout, and visuals whenever the surrounding layout or element size changes due to window resizing, dynamic visibility toggles, CSS modifications, or responsive design updates. Trigger recalculation of width and height, refresh the barcode rendering, and ensure it scales properly after programmatic size adjustments or DOM changes, enabling seamless resizing and layout adaptation without restarting or reinitializing the barcode component.
</div>

#### Example

    <div id="barcode" style="width: 300px; height: 150px;"></div>
    <script>
    $("#barcode").kendoBarcode({
      value: "FOO"
    });

    $("#barcode")
       .css("width", "600px")
       .data("kendoBarcode").resize();
    </script>

> **Important**
>
> If a fixed width is set using the [width](/api/javascript/dataviz/ui/barcode#configuration-width) option, the [resize](/api/javascript/dataviz/ui/barcode/methods/resize) method will not adjust the widget to match the size of the container. In this case, update both the container's width and the widget's width to the appropriate value.

#### Example

    <div id="barcode" style="width: 300px; height: 150px;"></div>
    <script>
    $("#barcode").kendoBarcode({
      value: "FOO",
      width: 300
    });

    $("#barcode").css("width", "600px");
    var barcode = $("#barcode").data("kendoBarcode");
    barcode.setOptions({ width: 600 });
    barcode.resize();
    </script>

#### Parameters

##### force `Boolean` *optional*

Defines whether the widget should proceed with resizing even if the element dimensions have not changed.

### svg

Returns the [SVG](https://www.w3.org/Graphics/SVG/) representation of the barcode. The returned string is a self-contained SVG document that can be used as is or converted to other formats using tools like [Inkscape](https://inkscape.org/en) and
[ImageMagick](https://www.imagemagick.org/). Both programs provide command-line interface suitable for server-side processing.

> This method is obsoleted by [exportSVG](/api/javascript/dataviz/ui/barcode/methods/exportsvg), but will remain fully functional.


<div class="meta-api-description">
Generate or retrieve a self-contained SVG string representing the barcode output for purposes such as exporting, saving, printing, or converting to other image formats; enable extraction of the barcode as a standalone SVG document suitable for use in server-side workflows, command-line tools, or graphic converters like Inkscape and ImageMagick, providing a flexible vector format export method that supports automated processing, embedding, or further manipulation in various environments.
</div>

#### Returns

`String` the SVG representation of the barcode.

#### Example - get the SVG representation of the barcode

    <div id="barcode"></div>
    <script>
    $("#barcode").kendoBarcode({
      value: "FOO",
      width: 300
    });
    var barcode = $("#barcode").data("kendoBarcode");
    var svg = barcode.svg();
	/* The result can be observed in the DevTools(F12) console of the browser. */
    console.log(svg); // displays the SVG string
    </script>

### value

Gets/Sets the value of the barcode.


<div class="meta-api-description">
Retrieve, read, access, get, or fetch the current barcode data or content programmatically, as well as update, set, change, modify, or overwrite the barcode value dynamically within the barcode component by using a method that returns the existing barcode value when called without parameters or updates the displayed barcode when called with a new input, supporting scenarios where developers need to control, configure, manipulate, or manage barcode information in applications, including resetting, refreshing, or syncing barcode content through code.
</div>

#### Example

    <div id="barcode"></div>
    <script>
    $('#barcode').kendoBarcode({
        width: 300,
        value: "123456"
    });
    // get a reference to the barcode widget
    var barcode = $("#barcode").data("kendoBarcode");

    // get the value of the barcode.
    var value = barcode.value();
	/* The result can be observed in the DevTools(F12) console of the browser. */
    console.log(value);

    // sets the value of the barcode and redraws it.
    barcode.value("1234567");
    </script>

#### Parameters

##### value `Number | String`

The value to set.

#### Returns

`String` The value of the barcode.
